WHPX Integration Plan
=====================

1. ~~**Abstract CPU Backend**~~
   - ~~Introduce a small interface so the emulator can switch between the existing
     dynamic recompiler and any new virtualization backend.~~
   - ~~Provide stubs that compile on all platforms.~~

2. ~~**Prototype WHPX Backend**~~
   - ~~Port the minimal WHPX initialisation logic from QEMU as a reference.~~
   - ~~Implement partition creation and basic VCPU management.~~
   - ~~Hook the backend behind a build-time option `--enable-whpx`.~~
   - ~~Basic memory mapping of guest RAM~~

3. **Incremental CPU Emulation Migration**
   - ~~Start executing basic instructions via WHPX while falling back to the
     interpreter/recompiler for unimplemented parts.~~
  - ~~Maintain feature parity and verify with existing regression tests.~~
- ~~Initial VCPU run stub integrated through `cpu_backend`.~~
  - ~~Synchronize general-purpose registers on each VM entry/exit~~
  - ~~Synchronize segment registers during VM transitions~~
  - ~~Handle WHPX exit reasons in the run loop~~

4. **Replace Dynamic Recompiler**
   - When all CPU features work reliably through WHPX, remove the old dynamic
     recompiler code.
   - Clean up configuration and documentation accordingly.

5. **Optimisation and Polish**
   - Profile performance, resolve any host/guest integration issues and ensure
     stable cross-version behaviour.
